jQuery插件开发

发表于 2017-11-26

简介

jQuery插件开发就是自定义封装一些方法，dom操作到jquery中，配合jQuery一起使用。

  • 插件推荐命名jquery.[插件名].js
  • 自定义插件必须在引入jQuery后

jQuery开发模式

jQuery主要有两种开发，类级别的插件开发，对象级别的插件开发。

类级别的插件开发

即给jQuery类添加静态方法，例如$.ajax()，$.trim()等。

jQuery.pluginName

添加全局函数

\$.extend(pluginName)

\$.extend()扩展jquery对象

命名空间

一般为了避免引入jQuery插件时，函数名或参数名冲突，会将方法封装到自定义的插件命名空间中。

对象级别的插件开发

这种模式的插件开发主要是对dom对象操作，使用\$(selector).pluginName()调用插件。

2种方式

开发方式有\$.fn.extend(myplugin),\$.fn.myplugin两种方式等价。

使用

支持链式调用

如果插件方法是不是获取数值，保持jquery调用用一个方法后能接着调用其他方法的特性，在插件内部返回dom的jquery对象。

初始化参数

定义插件时通常都会提供一些参数供调用者传入来初始化，增加插件灵活性。
可以使用$.extend()整合传入的参数和默认的参数。

面向对象开发

当编写一个复杂的插件时，代码量会很多，如果在插件内定义大量的变量、方法，会使代码变得杂乱，不方便维护，甚至会污染插件外部变量方法。这时就需要用面向对象的思维。

  • 把变量、方法定义到对象的原型属性中，使用方法时通过对象属性获取。
  • 使用data方法保存实例对象到dom中，避免重复实例化对象，调用插件先通过data = \$this.data(‘plugin_name’)获取对象，如果不存在则保存对象实例\$this.data(‘plugin_name’, (data = new pluginName(this, options)));
  • 定义获取值的方法时，通过字符串名获取插件内对象方法，如果调用插件传的参数 typeof option ===’string’为true，则调用data中的方法data[option].(data, args)，把获取到的值返回去。

\$.fn.extend(myplugin), \$.extend(), \$.fn.myplugin区别

$.fn 定义

所以\$.fn.extend(myplugin),\$.fn.myplugin都是对jQuery的原型进行扩展，jQuery类的实例可以使用该插件（即参数、函数），。

结论：

  1. \$.fn.extend(myplugin) 等价于 \$.prototype.extend(myplugin) 等价于\$.fn.myplugin。

  2. \$.fn.myplugin和\$.fn.extend(myplugin) 一般用于对象级别的插件开发。

  3. \$.extend() 一般用户类级别的开发。
