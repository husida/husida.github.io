mockjs模拟接口数据

发表于 2017-11-12

mockjs官网：http://mockjs.com/

简介

在前后端分离的环境下，前端的页面内容渲染需要依赖后台接口，而大多数情况下开发是前后端同时进行，就会出现后台接口没完成的时，前端只能写静态页面，对复杂数据的模拟则不是很方便。mockjs则可以更方便模拟真实接口数据，让前后端分离开发更方便。
mockjs的作用是生成随机数据，拦截 Ajax 请求。主要有以下功能：

  • 前后端分离，让前端攻城师独立于后端进行开发；
  • 增加单元测试的真实性，通过随机数据，模拟各种场景；
  • 开发无侵入，不需要修改既有代码，就可以拦截 Ajax 请求，返回模拟的响应数据；
  • 用法简单，符合直觉的接口；
  • 数据类型丰富，支持生成随机的文本、数字、布尔值、日期、邮箱、链接、图片、颜色等；
  • 方便扩展，支持支持扩展更多数据类型，支持自定义函数和正则。

语法规范

Mock.js 的语法规范包括两部分：

  1. 数据模板定义规范（Data Template Definition，DTD）
  2. 数据占位符定义规范（Data Placeholder Definition，DPD）

数据模板定义规范 DTD

由 3 部分构成：属性名name、生成规则rule、属性值value：

  • 属性名和生成规则用’|’隔开；
  • 生成规则根据属性值类型确定；
  • 属性值类型有String，Number，Boolean，Object，Array，Function，RegExp

数据占位符定义规范 DPD

占位符 只是在属性值字符串中占个位置，并不出现在最终的属性值中。
占位符 的格式为：

  1. 用 @ 来标识其后的字符串是 占位符。
  2. 占位符 引用的是 Mock.Random 中的方法。
  3. 通过 Mock.Random.extend() 来扩展自定义占位符。
  4. 占位符 也可以引用 数据模板 中的属性。
  5. 占位符 会优先引用 数据模板 中的属性。
  6. 占位符 支持 相对路径 和 绝对路径。

拦截 Ajax 请求

  1. Mock.mock( rurl?, rtype?, template|function( options ) ) 所带的参数都是可选的。
  2. 使用Mock.mock( template )参数是MockJs的语法模板时可以生产需要的一些接口数据。
  3. 使用rul，rtype参数，则可以配置拦截Ajax请求。
  4. 使用Mock.setup(settings )配置拦截 Ajax 请求时的行为。支持的配置项有：timeout。

Mock.mock()

Mock.mock( rurl?, rtype?, template|function( options ) )

  • rurl
    可选。
    表示需要拦截的 URL，可以是 URL 字符串或 URL 正则。例如 /\/domain\/list.json/、’/domian/list.json’。

  • rtype
    可选。
    表示需要拦截的 Ajax 请求类型。例如 GET、POST、PUT、DELETE 等。

  • template
    可选。
    表示数据模板，可以是对象或字符串。例如 { ‘data|1-10’:[{}] }、’@EMAIL’。

  • function(options)
    可选。
    表示用于生成响应数据的函数。拦截到ajax请求function将会执行并返回的数据。
    options
    指向本次请求的 Ajax 选项集，含有 url、type 和 body 三个属性，参见 XMLHttpRequest 规范。

Mock.setup()

Mock.setup( settings )，配置拦截 Ajax 请求时的行为。
支持的配置项有：timeout，拦截到请求时响应的延迟时间，值可以固定，也可以随机范围内，默认值是’10-100’。
